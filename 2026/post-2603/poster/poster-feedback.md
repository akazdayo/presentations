# main.typ レビュー

`main.typ` を見た限り、今のポスターに足りないのは「どこまで成功して、どこが未解決なのかを短時間で納得させる情報」です。背景や目的は十分ですが、評価と結論の見せ方がまだ弱いです。

## 優先度高

- 評価の基準が足りない  
  [結果](./main.typ) では「比較的良好」「一定程度再現」と書かれていますが、位置誤差、成功率、破綻条件、被験者数、試行数がなく、読者が「どこまでできたのか」を判断できません。遅延の平均値だけでは弱く、少なくとも評価指標と測定条件が必要です。

- タイトル近くの結論がない  
  タイトル直下から [主張](./main.typ) まで読むと「何をしたか」は分かりますが、「結局できたのか」は [結果](./main.typ) まで行かないと分かりません。タイトルの近くに「腰・脚の位置追従は可能だが、回転情報と遅延が課題」くらいの一文があると伝わり方がかなり変わります。

- 比較が文章だけで弱い  
  [比較](./main.typ) は物理トラッカーとの差を説明していますが、文章だけなので流し読みで拾いにくいです。`コスト / 装着性 / 位置精度 / 回転取得 / 遮蔽耐性 / 遅延` を並べた比較表が欲しいです。

## 優先度中

- 実験条件がまだ足りない  
  [手法](./main.typ) の実験条件には PC とカメラはありますが、カメラ位置、被写体との距離、照明、服装、動作パターン、VRChat を同時実行した条件、何回測ったかがありません。特に [結果](./main.typ) の `51.17 ms` と `85.18 ms` の解釈に必要です。

- 失敗例がない  
  [手法](./main.typ) のデモ画像は成功例としては良いですが、[結果](./main.typ) や [考察](./main.typ) で述べている「遮蔽」「速い動き」「回転情報不足」が視覚的に示されていません。破綻した姿勢や追従遅れの例を1枚でも入れると説得力が上がります。

- 新規性や貢献が明文化されていない  
  [目的](./main.typ) と [考察](./main.typ) はありますが、「既存手法に対して何を新たに示したか」が独立していないため、実装報告寄りに見えます。`本研究の貢献` のような短い整理があると研究として締まります。

## 優先度低

- QR の導線が弱い  
  今の QR は GitHub 向けですが、来場者向けにはデモ動画のほうが効果的です。加えて URL に `https://` がないので、読み取り環境によっては扱いが不安定になる可能性があります。

- 参考文献が少ない  
  [参考文献](./main.typ) は最低限で、MeTRAbs と VRChat OSC が中心です。比較対象の FBT 手法や、単眼姿勢推定の VR 応用例を 1-2 件入れると研究ポスターとしての厚みが出ます。

## 最優先で足すなら

1. 評価指標と測定条件
2. タイトル付近の結論一文
3. 物理トラッカーとの比較表
